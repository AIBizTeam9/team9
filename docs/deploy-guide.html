Vercel 가입
  1. https://vercel.com 접속
  2. Sign Up → Continue with GitHub 클릭
  3. GitHub 계정으로 로그인하면 가입 완료
2
GitHub 저장소 연결
  1. Vercel 대시보드에서 Add New... → Project
  2. Import Git Repository → AIBizTeam9/team9 선택
  3. Framework는 Next.js (자동 감지됨) 확인 후 Deploy 클릭
💡 저장소가 안 보이면 Adjust GitHub App Permissions를 클릭해서 AIBizTeam9 organization에 접근 권한을 추가하세요.

⚠️ 빌드 설정 확인 (중요!)

배포가 실패하면 빌드 설정을 확인해야 합니다.

프로젝트 → 왼쪽 메뉴 Settings → Build and Deployment
아래 값들이 맞는지 확인하세요:
항목값
Framework PresetNext.js
Build Commandnpm run build 또는 next build (기본값)
Output DirectoryNext.js default (비워두기)
Install Command기본값 (비워두기)
⚠️ Framework Preset이 Next.js가 아니면 빌드가 실패합니다! 반드시 Next.js로 설정하고, Output Directory는 Override하지 마세요.
3
Vercel 환경변수 설정

Supabase DB에 연결하기 위한 환경변수를 설정합니다.

Vercel 대시보드 → 프로젝트 클릭 → 왼쪽 메뉴 Settings → Environment Variables

아래 2개의 환경변수를 추가하세요:

Key Value
NEXT_PUBLIC_SUPABASE_URL 카카오톡으로 전달받은 값
NEXT_PUBLIC_SUPABASE_ANON_KEY 카카오톡으로 전달받은 값
📱 키 값은 동근이가 카카오톡으로 전달합니다. Key 이름은 위 표 그대로 복사하고, Value만 카톡에서 받은 값을 붙여넣으세요. 모든 팀원이 같은 DB를 공유합니다.

각 환경변수 추가할 때:

  1. Key에 변수 이름 입력 (예: NEXT_PUBLIC_SUPABASE_URL)
  2. Value에 카톡에서 받은 값 붙여넣기
  3. Environment는 모두 체크 (Production, Preview, Development)
  4. Save 클릭
4
내 브랜치 배포 연결 (Deploy Hook)

자기 브랜치에 push할 때마다 자동으로 배포되도록 webhook을 만듭니다.

Vercel 대시보드 → 프로젝트 → Settings → Git → 아래로 스크롤 → Deploy Hooks
  1. Name에 본인 이름 입력 (예: jiyun, jaerim, seokbin)
  2. Branch에 본인 브랜치 이름 입력 (예: jiyun)
  3. Create Hook 클릭
💡 이렇게 하면 본인 브랜치에 git push할 때마다 Vercel이 자동으로 빌드하고 배포합니다. 배포된 사이트는 Vercel 대시보드 → Deployments 에서 확인할 수 있어요.
5
로컬 개발 환경변수 설정 (바이브 코딩용)

Vercel에 배포할 때는 위에서 설정한 환경변수가 자동으로 적용되지만, 내 PC에서 개발할 때는 별도로 파일을 만들어야 합니다.

방법: .env.local 파일 만들기

프로젝트 최상위 폴더 (team9/)에 .env.local 파일을 만들고, 카카오톡으로 받은 내용을 통째로 붙여넣으세요.

# 카카오톡으로 받은 내용을 그대로 붙여넣으세요
NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbG...긴키값
⚠️ 동근이가 카카오톡으로 KEY=VALUE 뭉태기를 보내드립니다. 그 내용을 .env.local 파일에 그대로 붙여넣기만 하면 됩니다. 직접 값을 입력할 필요 없어요!

VS Code에서 파일 만드는 방법

  1. VS Code에서 프로젝트 폴더 열기
  2. 왼쪽 파일 탐색기에서 team9 폴더 우클릭 → New File
  3. 파일 이름: .env.local (앞에 점 필수!)
  4. 카카오톡에서 받은 내용 붙여넣기 → 저장 (Ctrl+S)
✅ 이 파일은 .gitignore에 등록되어 있어서 GitHub에 올라가지 않습니다. 안심하세요!

❓ FAQ
Q. 배포된 사이트 URL은 어디서 봐요?
Vercel 대시보드 → Deployments → 가장 최근 배포 클릭 → 상단에 URL이 표시됩니다.
Q. .env.local 파일이 GitHub에 올라가면 어떡해요?
.gitignore에 이미 등록되어 있어서 올라가지 않습니다. 만약 실수로 올라갔다면 동근이에게 알려주세요.
Q. 빌드가 실패하면?
Vercel 대시보드 → Deployments → 실패한 배포 클릭 → 로그를 확인하세요. 에러 메시지를 Copilot에게 보여주면 해결 방법을 알려줍니다.
Q. 환경변수를 바꿨는데 반영이 안 돼요
환경변수를 바꾼 후에는 다시 배포해야 적용됩니다. Deployments에서 가장 최근 배포의 ... 메뉴 → Redeploy를 클릭하세요.
Q. 같은 저장소를 여러 명이 연결해도 되나요?
네! 각자 본인의 Vercel 계정에서 같은 GitHub 저장소를 연결하면 됩니다. Deploy Hook을 각자 브랜치로 만들면 각자의 배포가 독립적으로 동작합니다.
