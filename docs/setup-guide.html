성공! 왼쪽에 app/, docs/, lib/ 폴더가 보이면 됩니다.
PART 3 — 내 브랜치로 이동

브랜치란?

브랜치 = 나만의 작업 공간. 각자 본인 이름의 브랜치에서 작업하면 다른 사람 코드와 충돌이 없습니다.

터미널 입력 내 브랜치 이동

VS Code 터미널(Ctrl + `)에서 본인 이름으로 입력:

git checkout -b 본인이름

예시: git checkout -b jiyun / git checkout -b jaerim / git checkout -b seokbin

항상 확인! VS Code 왼쪽 하단에 본인 이름이 표시되는지 확인하세요. main이면 안 됩니다.
PART 4 — GitHub Copilot 설정

Copilot 활성화

PART 2에서 GitHub 로그인을 했다면 Copilot도 바로 사용 가능합니다.

1

VS Code 상단 알림에서 “Use AI Features with Copilot for Free” 선택

2

Ctrl + Alt + I로 채팅창 열기

3

채팅창 상단에서 모드를 “Agent”로 선택

Agent 모드가 바이브 코딩의 핵심입니다. AI가 직접 파일을 만들고 수정합니다.
Free vs Pro: Free는 채팅 50회/월. 바이브 코딩을 본격적으로 하려면 Pro 무료 체험(30일)을 추천합니다.
PART 5 — 바이브 코딩 시작!

내 작업 시작하기 (핵심!)

각 팀원별로 할 일이 정리된 .md 파일이 있습니다. Copilot에게 이렇게 요청하세요:

docs/본인이름.md 파일을 읽고, 1단계부터 순서대로 진행해줘.

지윤: docs/jiyun.md 파일을 읽고, 1단계부터 순서대로 진행해줘.
재림: docs/jaerim.md 파일을 읽고, 1단계부터 순서대로 진행해줘.
석빈: docs/seokbin.md 파일을 읽고, 1단계부터 순서대로 진행해줘.

이 한 줄이면 AI가 알아서 페이지를 만들어줍니다. 단계별로 확인하면서 진행하세요.

바이브 코딩 팁

  • 마음에 안 들면? → “이 부분을 ~로 바꿔줘”
  • 에러가 나면? → 에러 메시지 복사해서 “이 에러 고쳐줘”
  • 결과 확인? → “개발 서버 실행해줘”
  • 한 번에 하나씩! → 너무 많은 걸 한꺼번에 요청하지 마세요
PART 6 — 저장하고 GitHub에 올리기

Commit & Push

작업이 끝나면 AI에게 저장을 요청하세요:

AI에게 요청 지금까지 만든 코드를 git에 커밋하고 push 해줘. 브랜치는 본인이름.

또는 VS Code GUI로 직접:

1

Ctrl + Shift + G로 소스 제어 열기

2

Changes 옆 + 버튼으로 파일 스테이징

3

메시지 입력 (예: “퀴즈 페이지 추가”) → Commit

4

Sync Changes 클릭 → GitHub에 업로드

처음 Push할 때 “브랜치를 게시하시겠습니까?” → OK를 누르세요.
PART 7 — PR (코드 반영 요청)

PR이란?

Pull Request = “내가 만든 코드를 팀 코드(main)에 넣어주세요”라는 요청입니다.

우리 팀은 최소 1명의 팀원이 확인(Approve)해야 main에 반영됩니다.

Push 완료
→
PR 생성
→
팀원이 확인
→
Approve
→
Merge!

GitHub 사이트 PR 만들기

1

GitHub 저장소 접속

2

상단 노란 배너의 “Compare & pull request” 클릭

3

제목 입력 (예: “퀴즈 페이지 추가”) → “Create pull request”

4

우측에서 Reviewers에 팀원 지정 → 카톡으로 리뷰 요청

PR 생성 후 코드를 더 수정하면? 같은 브랜치에 push하면 PR에 자동 반영됩니다. 새 PR을 만들 필요 없어요.

GitHub 사이트 팀원의 PR 리뷰하기

팀원이 “리뷰 부탁해”라고 하면:

1

PR 페이지 → “Files changed” 탭에서 변경 내용 확인

2

우측 상단 “Review changes” 클릭

3

“Approve” 선택 → 코멘트 작성 (예: “LGTM!”) → “Submit review”

코딩을 몰라도 리뷰할 수 있어요! “결과물이 잘 보인다”, “오타 있다” 같은 피드백도 훌륭한 리뷰입니다.

GitHub 사이트 Merge하기

1명 이상 Approve를 받으면:

1

PR 하단의 “Merge pull request” 클릭

2

“Confirm merge” 클릭 → 완료!

Merge 버튼이 비활성화? → 아직 Approve를 받지 못한 것입니다. 팀원에게 리뷰를 요청하세요.
PART 8 — Vercel 배포 (선택)

Vercel로 내 사이트 배포하기

코드를 push하면 자동으로 웹사이트가 만들어집니다. 처음 한 번만 설정하면 됩니다.

Vercel 사이트 초기 설정

1

vercel.com → Sign Up → Continue with GitHub

2

Add New → Project → AIBizTeam9/team9 선택

3

Framework Preset이 Next.js인지 확인 → Deploy

Framework Preset = Next.js 필수! “Others”로 되어 있으면 반드시 변경하세요. 안 바꾸면 빌드 실패합니다.

Vercel 사이트 내 브랜치를 도메인에 연결

기본 도메인이 내 브랜치의 최신 코드를 보여주도록 설정합니다:

1

Vercel 대시보드 → 프로젝트 → Settings → Domains

2

도메인 옆 Edit 클릭

3

Git Branch를 본인 브랜치 이름으로 변경 → Save

Vercel 사이트 환경변수 설정

Settings → Environment Variables에서 동근이가 카톡으로 보내는 값을 추가:

NEXT_PUBLIC_SUPABASE_URL = 카톡에서 받은 값 NEXT_PUBLIC_SUPABASE_ANON_KEY = 카톡에서 받은 값

Environment는 모두 체크 (Production, Preview, Development) → Save

Vercel 사이트 다른 사람에게 공유하기

Settings → Deployment Protection → Vercel Authentication을 Off로 변경하면 누구나 URL로 접속할 수 있습니다.

문제 해결

자주 겪는 문제

“git is not recognized”

→ Git 설치 후 VS Code를 껐다 다시 켜세요.

Push할 때 “rejected”

→ main 브랜치에서 push하면 거부됩니다. 본인 브랜치에서 해야 합니다.

git checkout 본인이름

Copilot 채팅이 안 열려요

→ VS Code를 최신 버전으로 업데이트하세요. (Help → Check for Updates)

Vercel에 여러 브랜치를 하나의 URL로 배포할 수 있나요?

→ 안됩니다. Vercel에서 하나의 도메인 = 하나의 브랜치입니다. 여러 브랜치를 하나의 URL에 자동 배포하는 건 불가능해요.
