# PREMORTEM.md

## Premortem: AI 기반 커리어 탐색 서비스 실패 시나리오 분석

Objective: 본 문서는 우리 팀의 서비스가 출시 후 실패했다고 가정하고, 그 원인을 사전에 분석하여 리스크를 줄이기 위함

---

## 1. 사용자 가치 부족 (Input 대비 Output 문제)

### 실패 시나리오
- 사용자는 서비스 초반 질문 단계에서 이탈한다.
- AI가 제공하는 커리어 추천이 지나치게 일반적이고 추상적이라 실질적인 도움이 되지 않는다.
- 사용자는 자신의 고민을 입력하는 것 자체에 부담을 느낀다.

### 원인
- 질문이 길고 피로도가 높음 (survey fatigue)
- LLM 결과가 "MBTI식 조언"처럼 느껴짐
- 개인적인 정보를 입력하는 것에 대한 심리적 장벽 존재

### 대응 전략
- 질문 수를 최소화하고 conversational UX로 자연스럽게 유도
- 단순 추천이 아닌 **구체적 액션 (예: 지원 가능한 채용 링크, 다음 행동 제안)** 제공
- 익명성 강조 및 데이터 저장 여부를 사용자에게 명확히 안내

---

## 2. 기술적 구현 리스크

### 실패 시나리오
- 채용 데이터 연동이 제대로 이루어지지 않아 서비스가 정적인 정보 제공에 그침
- GPT를 사용하는 것 이상의 차별화된 기능을 제공하지 못함

### 원인
- 채용 플랫폼 API 제한 또는 크롤링 어려움
- LLM 활용 구조가 단순 Q&A 수준에 머무름

### 대응 전략
- 초기 MVP 단계에서는 **외부 API 의존도를 낮추고 curated dataset 활용**
- 단순 질문 응답이 아닌 **구조화된 flow (진단 → 분석 → 추천)** 설계
- "AI 상담 경험" 자체를 차별화 포인트로 설계

---

## 3. 시장성 및 차별화 부족

### 실패 시나리오
- 기존 플랫폼 (LinkedIn, Wanted, GPT 등) 대비 차별성이 부족함
- 사용자가 한 번 사용하고 다시 돌아오지 않음

### 원인
- 서비스가 “재미있는 테스트” 수준으로 인식됨
- 지속적인 사용 유인을 제공하지 못함

### 대응 전략
- 단순 추천이 아닌 **"나를 이해하는 경험" (Self-reflection loop)** 제공
- 결과를 저장하고 발전시키는 **Career Journey Tracking 기능** 고려
- 주기적으로 리마인드/업데이트를 유도하는 구조 설계

---

## 4. 사용자 행동 및 심리 리스크

### 실패 시나리오
- 사용자가 자신의 커리어 고민을 깊이 있게 입력하지 않음
- 결과를 신뢰하지 않거나 가볍게 소비하고 종료

### 원인
- AI에 대한 신뢰 부족
- 개인 정보 입력에 대한 불안감
- 결과의 개인화 수준이 낮음

### 대응 전략
- 결과에 대한 **논리 설명 (Why this recommendation?) 제공**
- 사용자의 input을 반영했다는 느낌을 주는 personalization 강화
- 간단한 input → 점진적 deep dive 구조 설계

---

## 5. 핵심 요약

본 서비스가 실패하는 가장 큰 이유는 다음 세 가지로 요약된다:

1. 사용자에게 즉각적이고 실질적인 **가치 제공 실패**
2. 기존 AI 및 커리어 서비스 대비 **차별화 부족**
3. 지속적으로 사용하게 만드는 **유인 구조 부재**

---

## 6. 핵심 대응 방향

- "재미있는 서비스"가 아닌 **"의사결정을 도와주는 도구"**로 포지셔닝
- 단순 AI 활용이 아닌 **경험 설계 중심 서비스 개발**
- 결과는 반드시 **Actionable (행동으로 이어질 수 있는 형태)** 로 제공

---

## 결론

이 Premortem을 통해 확인한 가장 중요한 인사이트는,  
우리 서비스의 성공 여부는 기술이 아니라 **사용자에게 얼마나 '실제 도움이 되는 경험'을 제공하느냐**에 달려 있다는 점
