AI와 대화하며 코드를 만드는 바이브 코딩을 시작하세요.

VS Code + Git + GitHub Copilot — AIBizTeam9

바이브 코딩(Vibe Coding)이란?

코드를 직접 작성하는 대신, AI에게 자연어(한국어)로 원하는 것을 설명하면 AI가 코드를 대신 생성해주는 방식입니다.

이 가이드에서는 GitHub Copilot의 Agent 모드를 사용합니다. Copilot 채팅창에 한국어로 원하는 기능을 설명하면, AI가 코드를 생성하고 파일을 만들어줍니다.

예시: "로그인 페이지 만들어줘" → Copilot이 알아서 코드 파일을 생성!

PART 1 — 프로그램 설치
1

📦 VS Code 설치

VS Code(Visual Studio Code)는 마이크로소프트에서 만든 무료 코드 에디터입니다. 바이브 코딩의 작업 공간이 됩니다.

VS Code 다운로드
  1. 위 버튼을 클릭하여 다운로드 페이지로 이동합니다.
  2. "Download for Windows" 버튼을 클릭합니다.
  3. 다운로드된 설치 파일(VSCodeUserSetup-x64-*.exe)을 실행합니다.
  4. 설치 화면에서 "다음(Next)"을 계속 눌러 설치를 완료합니다.
체크 필수! 설치 도중 "추가 작업 선택" 화면이 나오면 다음 항목들을 반드시 체크하세요:
  • "PATH에 추가" — 터미널에서 code 명령어를 사용할 수 있게 됩니다
  • "Code(으)로 열기" 작업을 추가 — 폴더를 우클릭하여 VS Code로 열 수 있습니다
2

🔧 Git 설치

Git은 코드의 변경 이력을 관리하고, 팀원들과 코드를 공유하는 도구입니다.

Git 다운로드
  1. 위 버튼을 클릭하여 다운로드 페이지로 이동합니다.
  2. "Download for Windows" 버튼을 클릭합니다.
  3. 다운로드된 설치 파일(Git-*-64-bit.exe)을 실행합니다.
  4. 설치 화면에서 모든 옵션을 기본값 그대로 두고 "Next"를 눌러 설치합니다.
중요! Git 설치가 끝나면 VS Code를 완전히 종료했다가 다시 실행하세요. 그래야 VS Code가 Git을 인식합니다.
PART 2 — GitHub 로그인 & 프로젝트 가져오기
3

📂 GitHub 로그인 & 프로젝트 Clone (복사)

VS Code에서 GitHub에 로그인하면 Git 설정이 자동으로 완료됩니다. 별도의 터미널 명령어가 필요 없습니다!

Step 1: VS Code에서 GitHub 로그인

  1. VS Code를 실행합니다.
  2. 좌측 하단의 사람 아이콘을 클릭합니다.
  3. "Sign in with GitHub to use GitHub Copilot"을 선택합니다.
  4. 브라우저가 열리면 GitHub 계정으로 로그인하고 "Authorize"를 클릭합니다.
  5. VS Code로 자동 복귀됩니다.
로그인 완료! GitHub 로그인이 되면 Git 인증 정보가 자동으로 설정됩니다. push/pull 할 때 따로 비밀번호를 입력할 필요가 없습니다.

Step 2: 프로젝트 Clone

  1. VS Code 시작 화면에서 "Clone Git Repository..."를 클릭합니다.
    (또는 Ctrl + Shift + P → Git: Clone 입력)
  2. 상단 입력창에 아래 주소를 붙여넣습니다:
https://github.com/AIBizTeam9/team9.git
  1. Enter를 누릅니다.
  2. 프로젝트를 저장할 폴더 위치를 선택합니다. (바탕화면, 문서 등 원하는 곳)
  3. "Select as Repository Destination"을 클릭합니다.
  4. 클론이 완료되면 "Open"을 눌러 프로젝트를 엽니다.
성공 확인! VS Code 왼쪽 Explorer에 app/, backend/, docs/ 폴더가 보이면 정상입니다.
커밋할 때 이름/이메일 오류가 나는 경우
드물게 Git이 사용자 정보를 요청할 수 있습니다. 그때만 터미널(Ctrl + `)에서 아래 두 줄을 입력하세요:
git config --global user.name "본인GitHub사용자명"
git config --global user.email "본인GitHub이메일"
PART 3 — 내 이름으로 브랜치 만들기
4

🌴 브랜치(Branch)란?

브랜치는 나만의 작업 공간입니다. 각자 본인 이름의 브랜치에서 작업하면, 다른 사람의 코드와 충돌 없이 자유롭게 바이브 코딩할 수 있습니다.

main (원본)
→
jiyun (지윤의 작업)
main (원본)
→
dongkeun (동근의 작업)

내 브랜치 만들고 이동하기

VS Code 터미널에서 아래 명령어를 입력하세요:

# "본인이름" 을 자신의 이름으로 바꾸세요 (영어, 소문자)
git checkout -b 본인이름
예시:
git checkout -b jiyun — 지윤
git checkout -b jaerim — 재림
git checkout -b dongkeun — 동근
git checkout -b seokbin — 석빈

브랜치 확인하기

VS Code 왼쪽 하단을 보면 현재 브랜치 이름이 표시됩니다. 본인 이름이 보이면 성공!

또는 터미널에서 확인:

git branch # * 표시가 현재 브랜치
주의! 항상 본인 브랜치에서 작업하세요. main 브랜치에서는 직접 작업하지 않습니다.
VS Code 왼쪽 하단에 본인 이름이 표시되는지 확인하는 습관을 들이세요.
PART 4 — GitHub Copilot 설정
5

🤖 GitHub Copilot 활성화

GitHub Copilot은 VS Code 안에서 AI와 대화하며 코드를 생성하는 도구입니다. 바이브 코딩의 핵심!

Step 3에서 이미 GitHub 로그인을 했다면, Copilot도 바로 사용 가능합니다.

Copilot 무료 플랜 활성화

  1. VS Code 상단에 알림이 뜨면 "Use AI Features with Copilot for Free"를 선택합니다.
  2. 알림이 안 보이면, 상단 타이틀바의 Copilot 아이콘을 클릭해보세요.
Copilot 요금제 안내:
  • Free: 월 2,000회 코드 자동완성 + 50회 채팅 (무료)
  • Pro ($10/월): 무제한 코드 자동완성 + 무제한 채팅 (30일 무료 체험 가능)

바이브 코딩을 본격적으로 하려면 Pro 무료 체험을 추천합니다. Free는 채팅 50회가 금방 소진됩니다.

Copilot 채팅창 열기

단축키 Ctrl + Alt + I를 눌러 Copilot 채팅창을 엽니다.

또는 VS Code 상단 타이틀바의 Copilot 아이콘을 클릭합니다.

성공! 채팅창이 열리고 메시지를 입력할 수 있으면 Copilot 설정이 완료된 것입니다.
PART 5 — 바이브 코딩 시작!
6

✨ Copilot Agent 모드로 바이브 코딩하기

이제 AI와 대화하며 코드를 만들 준비가 되었습니다!

Agent 모드 선택

  1. Copilot 채팅창을 엽니다: Ctrl + Alt + I
  2. 채팅창 상단의 모드 선택 드롭다운에서 "Agent"를 선택합니다.
Agent 모드란? 일반 채팅과 달리, Agent 모드는 AI가 직접 파일을 생성/수정하고, 터미널 명령어를 실행할 수 있습니다. 바이브 코딩에 최적화된 모드입니다.

바이브 코딩 방법

채팅창에 한국어로 원하는 것을 자연스럽게 설명하면 됩니다:

프롬프트 예시 1 - 페이지 만들기
"app 폴더 안에 퀴즈 페이지를 만들어줘. 사용자가 5개의 질문에 답하면 결과 페이지로 넘어가는 구조로 해줘."
프롬프트 예시 2 - 디자인 수정
"이 페이지 배경을 어두운 색으로 바꾸고, 버튼을 둥글게 만들어줘."
프롬프트 예시 3 - 기능 추가
"사용자가 퀴즈 답변을 제출하면 로딩 애니메이션을 보여주고, 3초 후에 결과 페이지로 이동하게 해줘."
프롬프트 예시 4 - 에러 해결
"터미널에 이런 에러가 나와: [에러 메시지 붙여넣기]. 이거 고쳐줘."

바이브 코딩 핵심 팁

  • 구체적으로 설명하세요: "좋은 페이지 만들어줘" 보다 "카드 형태로 3개의 옵션을 보여주는 선택 페이지"가 훨씬 좋은 결과를 냅니다.
  • 한 번에 하나씩: 한 번에 너무 많은 것을 요청하지 마세요. 기능 하나씩 차근차근 추가하세요.
  • 결과를 확인하세요: AI가 코드를 생성하면 미리보기로 결과를 확인하고, 마음에 안 드는 부분을 다시 요청하세요.
  • 변경 사항을 수락하세요: Agent가 파일을 수정하면 diff(변경 내용)가 표시됩니다. 내용이 괜찮으면 "Accept"를 클릭하세요.
  • 되돌리기: 마음에 안 들면 "Discard"를 클릭하거나, Ctrl + Z로 되돌릴 수 있습니다.
PART 6 — 작업 저장하고 GitHub에 올리기
7

💾 Git으로 작업 저장하기 (Commit & Push)

바이브 코딩으로 코드를 만들었으면 주기적으로 저장(commit)하고 GitHub에 올려(push)야 합니다. 중간 저장이라고 생각하세요!

방법 A: VS Code GUI 사용 (초보자 추천!)

  1. VS Code 왼쪽 사이드바에서 소스 제어(Source Control) 아이콘을 클릭합니다. (분기 모양 아이콘, 또는 Ctrl + Shift + G)
  2. 변경된 파일 목록이 보입니다. "Changes" 옆의 + 버튼을 클릭하여 모든 변경 파일을 스테이징합니다.
  3. 상단 입력창에 변경 내용 설명을 입력합니다. (예: "퀴즈 페이지 추가")
  4. "Commit" 버튼 ✓을 클릭합니다.
  5. "Sync Changes" 버튼을 클릭하여 GitHub에 업로드합니다.
처음 Push할 때 "브랜치를 게시(publish)하시겠습니까?" 라는 메시지가 뜹니다. "OK"를 눌러주세요. GitHub에 본인 브랜치가 처음 생성되는 것입니다.

방법 B: 터미널 사용

# 1. 변경된 파일 확인
git status

# 2. 모든 변경 파일을 스테이징
git add .

# 3. 변경 내용 저장 (커밋 메시지 작성)
git commit -m "퀴즈 페이지 추가"

# 4. GitHub에 올리기 (처음 한 번만 -u 옵션 필요)
git push -u origin 본인이름
"본인이름"을 본인 브랜치 이름으로 바꾸세요!
예시: git push -u origin jiyun

두 번째 push부터는 간단히 git push만 입력하면 됩니다.

PART 7 — Vercel 배포 (내 브랜치 → 웹사이트)
9

▲ Vercel 배포 설정하기

코드를 push하면 자동으로 웹사이트에 반영됩니다. 처음 한 번만 설정하면 됩니다.

Step 1: Vercel 가입

  1. https://vercel.com 접속
  2. "Sign Up" → "Continue with GitHub" 클릭
  3. GitHub 계정으로 로그인하면 가입 완료

Step 2: 프로젝트 연결

  1. Vercel 대시보드 → Add New... → Project
  2. Import Git Repository → AIBizTeam9/team9 선택
  3. 저장소가 안 보이면 "Adjust GitHub App Permissions" 클릭해서 권한 추가
Framework Preset = Next.js 필수!
Deploy 버튼 누르기 전에 반드시 확인하세요:
  • Framework Preset: Next.js (자동 감지가 안 되면 직접 선택)
  • Build Command: 기본값 그대로 (비워두기)
  • Output Directory: 기본값 그대로 (비워두기)

만약 "Others"로 되어 있으면 반드시 Next.js로 변경해야 합니다. 안 바꾸면 빌드가 실패합니다!

설정 확인 후 "Deploy" 클릭!

Step 3: Production Branch를 내 브랜치로 변경

Vercel은 기본적으로 main 브랜치를 Production으로 배포합니다. 하지만 우리는 각자 브랜치에서 개발 중이므로, 본인 브랜치를 Production으로 설정해야 합니다.

  1. Vercel 대시보드 → 프로젝트 → Settings
  2. 왼쪽 메뉴에서 Git 클릭
  3. Production Branch 항목을 찾습니다
  4. main을 본인 브랜치 이름으로 변경합니다
예시:
지윤 → jiyun
재림 → jaerim
동근 → dongkeun
석빈 → seokbin
왜 바꿔야 하나요? 이 설정을 안 바꾸면 "No Production Deployment" 메시지가 뜨고, 사이트 URL(your-project.vercel.app)로 접속이 안 됩니다. 지금은 각자 개발 중간단계를 확인하는 용도이므로 본인 브랜치를 Production으로 설정하세요.

Step 4: 빌드 설정 확인

빌드 설정 재확인 경로:
프로젝트 → 왼쪽 메뉴 Settings → Build and Deployment

여기서 Framework Preset이 Next.js인지 확인하세요. "Others"로 되어 있으면 변경 후 Redeploy!

Step 5: 환경변수 설정

Vercel 대시보드 → 프로젝트 → Settings → Environment Variables

동근이가 카카오톡으로 보내는 KEY=VALUE 값을 아래 형식으로 추가하세요:

NEXT_PUBLIC_SUPABASE_URL = 카톡에서 받은 값
NEXT_PUBLIC_SUPABASE_ANON_KEY = 카톡에서 받은 값

Environment는 모두 체크 (Production, Preview, Development) → Save

Step 6: Deploy Hook (자동 배포)

Settings → Git → 스크롤 → Deploy Hooks

  1. Name: 본인 이름 (예: jiyun)
  2. Branch: 본인 브랜치 이름 (예: jiyun)
  3. "Create Hook" 클릭
완료! 이제 git push하면 Vercel이 자동으로 빌드하고 배포합니다.
배포된 URL은 Vercel 대시보드 → Deployments에서 확인하세요.

Step 7: 로컬 환경변수 (.env.local)

로컬에서 개발할 때도 환경변수가 필요합니다.

  1. 프로젝트 폴더(team9/)에 .env.local 파일 생성
  2. 동근이가 카톡으로 보내는 KEY=VALUE 내용을 통째로 붙여넣기
  3. 저장 (Ctrl+S)
이 파일은 .gitignore에 등록되어 있어서 GitHub에 올라가지 않습니다. 안심하세요!
자주 겪는 문제 해결

😕 이런 에러가 나왔어요!

"git is not recognized" / "git을 찾을 수 없습니다"

→ Git 설치 후 VS Code를 완전히 종료했다가 다시 실행하세요.

"Please tell me who you are" / "사용자 정보를 설정해주세요"
